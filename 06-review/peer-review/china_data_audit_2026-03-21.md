**审计日期**: 2026-03-21
**审计人**: data-analyst
**审计范围**: Urban Q 项目中所有中国相关数据的完整来源链
**审计方法**: 逐一阅读数据生成脚本源代码、输出 CSV 文件、来源文档，追溯每个变量从原始来源到最终分析变量的完整路径

## 一、总体判断

**整体风险等级: 中等偏高**

项目存在一个结构性问题：多层数据中，真正从外部文件（权威机构发布的原始数据集）直接读取的比例有限，大量"真实数据"实际上是**脚本内硬编码**的数字，声称来源于统计年鉴但缺少可追溯的自动化下载/验证链路。硬编码数字经抽查基本合理，但无法排除手动录入错误，也不符合 Nature 系列期刊对数据可复现性的要求。

## 二、数据集逐一审计

### 数据集 1：国家级面板数据 (china_national_real_data.csv)

| 项目 | 详情 |
|------|------|
| 生成脚本 | `03-analysis/scripts/40_china_real_data.py` |
| 输出文件 | `02-data/raw/china_national_real_data.csv` |
| 覆盖范围 | 1990-2023年, 34行 x 36列 |
| 来源文档 | `02-data/raw/china_data_sources.md` |

**数据生成机制**:

脚本采用两层策略：
1. **策略1 (NBS API)**: 尝试通过 `data.stats.gov.cn` API 抓取数据。代码结构完整，包含请求头、反爬处理、数据验证逻辑。但在实际运行中，NBS API 通常因境外网络限制或反爬机制而失败。
2. **策略2 (统计年鉴硬编码)**: API 失败后的 fallback。所有 GDP、人口、投资、销售额、竣工面积数据均以 Python dict 字面量形式硬编码在 `build_yearbook_data()` 函数中。

**实际执行路径**: 根据 `china_national_real_data.csv` 中 `data_source` 列全部标记为 `YEARBOOK`，可确认实际使用的是**策略2（硬编码）**。

**抽查验证**:

| 变量 | 年份 | 脚本中的值 | 预期真实值 | 是否一致 |
|------|------|-----------|-----------|---------|
| GDP (亿元) | 2023 | 1,260,582.0 | 1,260,582 (统计公报) | 是 |
| GDP (亿元) | 2010 | 412,119.3 | 412,119.3 (统计年鉴) | 是 |
| 城镇化率 (%) | 2023 | 66.16 | 66.16 (统计公报) | 是 |
| 城镇化率 (%) | 2020 | 63.89 | 63.89 (七普) | 是 |
| 房地产投资 (亿元) | 2023 | 110,913.0 | 110,913 (统计公报) | 是 |
| 商品房销售额 (亿元) | 2021 | 181,930.0 | 181,930.0 (统计年鉴) | 是 |

**补充数据**: World Bank 数据从 `02-data/processed/world_bank_usable_panel.csv` 读取（该文件由 `20_world_bank_data.py` 通过 WB API 下载生成），作为国际可比口径补充。

**分类: C 类 (不能直接用于论文)**

理由：数据数值经抽查基本准确，但核心问题在于：
- 没有可追溯的自动化获取链路（NBS API 未成功执行）
- 硬编码数据无法被第三方独立复现
- 存在手动录入错误的风险（34年 x 6组变量 = 约200个数字手动输入）
- 来源文档写得很好但实质上是"事后补注"而非数据获取过程的自动记录

### 数据集 2：省级面板数据 (china_provincial_real_data.csv)

| 项目 | 详情 |
|------|------|
| 生成脚本 | `03-analysis/scripts/41_china_provincial_data.py` |
| 输出文件 | `02-data/raw/china_provincial_real_data.csv` |
| 覆盖范围 | 31省 x 19年 (2005-2023) = 589行 |
| 来源文档 | `02-data/raw/china_provincial_sources.md` |

**数据生成机制**:

与国家级脚本类似的双策略架构：
1. **策略1 (NBS 分省 API)**: 尝试通过 `fsnd` 数据库代码访问分省年度数据。实际同样因 API 不可用而失败。
2. **策略2 (硬编码 + 线性插值)**: 对 **5 个关键年份** (2005, 2010, 2015, 2019, 2023) 的 31 省 x 4 个变量 (GDP, 城镇化率, 三产比重, FAI) 进行硬编码。中间年份全部采用线性插值。

**声称的 "5个关键年份真实值 + 线性插值" 是否属实**: **部分属实**。

- CSV 文件中确实有 `data_type` 列，`actual` 标记 155 行 (26.3%), `interpolated` 标记 434 行 (73.7%)
- 5 个关键年份的数据确实硬编码于脚本中
- 但 **这些硬编码数字本身的验证链路缺失**

**抽查验证**:

| 省份 | 年份 | 变量 | 脚本值 | 预期值 | 是否一致 |
|------|------|------|--------|--------|---------|
| 广东 | 2023 | GDP (亿元) | 135,673.2 | 135,673.16 (粤统计公报) | 是 |
| 北京 | 2010 | GDP (亿元) | 14,113.6 | 14,113.58 (统计年鉴2011) | 是 |
| 西藏 | 2023 | GDP (亿元) | 2,392.7 | 2,392.67 (藏统计公报) | 是 |
| 上海 | 2023 | 城镇化率 (%) | 89.3 | 89.3 (第七次普查修正) | 是 |

**关键问题**:
1. 73.7% 的数据点为线性插值，对于 GDP 等非线性增长变量，线性插值会引入系统偏差
2. 2020-2023年固定资产投资绝对值全部为缺失（因统计局不再公布），这意味着省级 FAI/GDP 比率在近年不可用
3. 2023年 FAI 数据在代码中被强制设为 NaN（371行），即使原始硬编码中有部分省份数据

**分类: C 类 (不能直接用于论文)**

理由：同国家级数据问题 + 73.7% 插值比例过高。如果仅使用 5 个关键年份的截面数据，可提升至 B 类。

### 数据集 3：国家级 Urban Q 计算 (china_urban_q_real_data.csv)

| 项目 | 详情 |
|------|------|
| 生成脚本 | `03-analysis/scripts/50_china_urban_q_real.py` |
| 数据输入来源 | six-curves 项目的原始 CSV 文件 |

**数据流向**: 这是项目中数据质量最高的一环。

```
six-curves/02-data/raw/ 中的 CSV 文件
  -> c5_residential_price_NBS_1998-2024.csv
  -> c5_land_transfer_revenue_MOF_1999-2024.csv
  -> c6_real_estate_investment_NBS_1987-2024.csv
  -> c6_infrastructure_investment_NBS_1990-2024.csv
  -> c3_new_construction_starts_NBS_1985-2024.csv
  -> background_gdp_NBS_1978-2024.csv
  -> c1_urbanization_rate_NBS_1949-2024.csv
six-curves/02-data/processed/six_curves_master_dataset_processed.csv
  -> 50_china_urban_q_real.py 读取上述文件
  -> 计算 V1/V2/V3 (市场价值三口径) 和 K1/K2/K3 (资本存量三口径)
  -> 输出 Urban Q 时序
```

**V1/K1/K2 等变量的具体来源**:

- **V1** = residential_price (来自 c5 CSV) x housing_stock_pim (永续盘存法，基于 c3 竣工面积)
- **V2** = 累计商品房销售额 (来自 c5 CSV cumsum)
- **V3** = V1 + 累计土地出让收入 + 基础设施存量
- **K1** = FAI 资本存量（永续盘存法，折旧率 5%）
- **K2** = 房地产投资资本存量（折旧率 3%）

**关键问题**: 虽然该脚本从外部 CSV 读取数据（好），但追根溯源，这些 six-curves 项目的 CSV 文件本身也需要审计。

**分类: B 类 (需谨慎使用)**

理由：
- 数据从独立 CSV 文件读取（可追溯性好）
- 每个 CSV 均有 source_note 列标注具体年鉴版本
- 但 CSV 文件的生成过程不详（可能也是手动编制）
- 派生指标依赖折旧率假设和永续盘存法基期假设

### 数据集 5：城市面板数据

| 项目 | 详情 |
|------|------|
| 生成脚本 | `03-analysis/scripts/51_city_panel_real.py` |
| 核心输入 | 5 个外部数据文件 |

**300城市 x 34年的大面板是怎么来的**:

该面板不是凭空生成的。核心数据来源是：

| 数据源 | 路径 | 格式 | 性质 |
|--------|------|------|------|
| 中国城市数据库6.0版 | `six-curves/.../中国城市数据库6.0版.xlsx` (26.9MB) | Excel, "线性插值" sheet | **外部购买的商业数据库** (马克数据网)，包含 300+ 城市 x 214 列 |
| 全国地级市国民经济指标 | `six-curves/.../全国地级市2000-2023年国民经济指标.xlsx` (6.4MB) | Excel | **外部购买数据** |
| 58同城房价 | `six-curves/.../58同城房价（2010-2024）.xlsx` (135KB) | Excel | **外部购买数据** (含 .dta 格式) |
| 安居客房价 | `six-curves/.../安居客房价（2015-2024）.xlsx` (215KB) | Excel | **外部购买数据** |
| 地级市债务数据 | `six-curves/.../地级市/整合.csv` (519KB) | CSV | **外部购买数据** (2006-2023) |

**关键发现**:

1. **数据确实存在于文件系统中** -- 所有 5 个外部数据文件均在指定路径存在，文件大小合理
2. **脚本通过 `pd.read_excel()` 和 `pd.read_csv()` 直接读取** -- 而非脚本内构造合成数据
3. **城市数据库 6.0 版**: 26.9MB 的 Excel 文件，包含"线性插值"sheet，说明购买时已含缺失值处理版本
4. **房价数据**: 58同城覆盖 2010-2024，安居客覆盖 2015-2024，两源交叉验证取均值
5. **债务数据**: 包含 2006-2021 年的逐年子目录 + 2022/2023 单独 Excel

**数据处理流程**:
```
中国城市数据库6.0 (300+城市 x 214列)
  + 58同城房价 (365城市 x 15年)
  + 安居客房价 (补充验证)
  + 地级市债务 (2006-2023)
  -> 按 city_clean + year 合并
  -> 人口序列修正 (2020七普比率校准)
  -> FAI 缺失值估算 (2017后用 FAI/GDP ratio x GDP)
  -> 永续盘存法计算资本存量
  -> Urban Q = V/K
```

**分类: B 类 (需谨慎使用)**

理由：
- 核心数据从外部购买的商业数据库读取（非脚本生成），来源清晰
- 商业数据库（马克数据网）在国内学术界广泛使用，但 Nature 审稿人可能不熟悉
- 2017年后 FAI 通过 ratio 估算引入误差
- 人口序列通过 2020 普查比率校准，方法合理但引入假设
- 房价仅覆盖 2010-2024，之前年份缺失

## 五、需要替换/补充的数据清单

### 优先级1 (必须在投稿前完成)

1. **国家级面板 (C->B)**: 用 NBS 统计年鉴的原始 Excel 文件替代硬编码
   - 方案: 从 https://www.stats.gov.cn/sj/ndsj/ 下载历年统计年鉴 Excel 表格，编写解析脚本自动提取
   - 或: 使用 CEIC、WIND 等商业数据库导出，保留导出记录
   - 工作量: 约1天

2. **省级面板 (C->B)**: 同上，使用分省年度数据原始 Excel
   - 方案: 从 NBS 官网下载分省年度 Excel，或使用 CEIC/WIND
   - 额外: 降低插值比例，补充更多关键年份的真实数据（如 2000, 2008, 2012, 2016, 2020, 2022）
   - 工作量: 约2天

3. **补充数据获取日志**: 为 six-curves 的 11 个 CSV 文件补充数据获取记录
   - 方案: 在 `02-data/raw/` 中添加 `data_provenance.md`，记录每个文件的原始 Excel 来源路径和下载日期
   - 工作量: 约0.5天

### 优先级2 (建议在审稿前完成)

4. **FAI 2017-2023 缺口**: 城市级和省级固定资产投资在 2017 年后的缺失是系统性问题
   - 方案: 使用 Wind 或 CEIC 获取"固定资产投资完成额（不含农户）"的城市级数据，或使用省级统计公报补充
   - 备选: 在论文中明确说明 2017 后的投资数据为估算值，并在敏感性分析中测试不同估算方法

5. **房价数据时间覆盖**: 城市级房价仅覆盖 2010-2024，之前无房价数据
   - 方案: 补充 NBS 城市级"商品房销售均价"数据（2000-2009），从《中国房地产统计年鉴》获取
   - 影响: 限制了城市级 Urban Q 的时间跨度

6. **人口数据一致性**: 城市级常住人口在 2020 年七普之前大量缺失，用户籍人口 x 比率估算
   - 方案: 在论文中披露，并在敏感性分析中测试只用 2020 后数据

### 优先级3 (锦上添花)

7. **交叉验证脚本**: 编写自动化脚本对比硬编码数字与原始 Excel 数字
8. **数据版本控制**: 为关键数据文件建立校验和 (SHA-256)
9. **Supplementary Data**: 准备 Nature 要求的 Source Data 文件，包含所有原始数据和处理代码
